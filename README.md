## 📌 프로젝트 소개

**대한민국 밤 산책**은 경복궁 별빛야행 축제를 소개하고 홍보하기 위해 제작한 웹 프로젝트입니다.  
메인 축제 소개뿐 아니라 유사한 축제 정보도 함께 탐색할 수 있도록 구성했으며,  
`Three.js`와 `Blender`를 활용해 보다 생동감 있는 사용자 경험을 제공하고자 했습니다.

## ✨ 주요 구현 내용

### 1. 자주하는 질문(FAQ)
- **검색 기능**
  - 검색어 입력 후 버튼 클릭뿐 아니라 `Enter` 키로도 검색이 가능하도록 구현했습니다.
  - 검색 결과에 따라 리스트 개수가 줄어들면, 페이지네이션 개수도 자동으로 변경되도록 처리했습니다.

- **토글 기능**
  - 질문 클릭 시, 클릭한 항목의 `id`와 데이터 `id`를 비교해 해당 항목만 열리고 닫히도록 구현했습니다.

![FAQ 화면](https://github.com/user-attachments/assets/855b1747-7e65-4cba-85fa-ad933f730c6c)

### 2. 마이페이지 - 1:1 문의 메인
> 마이페이지는 백엔드 서버 연결이 필요한 영역이라 이미지로 대체했습니다.

- 로그인한 사용자와 동일한 아이디를 가진 문의 내역만 출력되도록 구현했습니다.
- 리스트 클릭 시 상세 페이지로 이동해 내용을 확인할 수 있도록 구성했습니다.
- 페이지네이션 컴포넌트를 적용해 한 페이지당 10개씩 출력되도록 했습니다.
- 최신 문의가 상단에 보이도록 **역순 정렬**해 사용자 편의성을 높였습니다.

![1:1 문의 메인](https://github.com/user-attachments/assets/1b434c0e-c311-4910-b851-980077b3d6eb)

### 3. 마이페이지 - 1:1 문의 상세 페이지
- 목록 버튼을 추가해 사용자가 다시 문의 메인 페이지로 쉽게 이동할 수 있도록 했습니다.
- 수정 / 삭제 버튼을 통해 각 문의 항목을 직접 관리할 수 있도록 구현했습니다.

- 수정 완료 / 취소 버튼 클릭 시 알림이 노출되도록 함수를 작성했습니다.

### 5. 마이페이지 - 1:1 문의 작성 페이지
- 메인 리스트의 글쓰기 버튼을 통해 문의 작성 페이지로 이동할 수 있도록 구현했습니다.
- 작성 완료 시 알림이 노출되고, 이후 메인 리스트 페이지로 이동하도록 처리했습니다.
- 작성 후 메인으로 돌아왔을 때 새 문의 내역이 최상단에 보이도록 구성했습니다.

### 6. 기타 구현
- `react-paginate` 라이브러리를 사용해 여러 페이지에서 재사용 가능한 **공용 페이지네이션 컴포넌트**를 제작했습니다.
- 전체 페이지를 반응형으로 구현해 다양한 기기에서도 일관된 사용자 경험을 제공할 수 있도록 했습니다.

## 💭 트러블슈팅

### 1. 데이터 전달 구조 문제
초기에는 `router`를 사용해 페이지를 이동시키면서, 필요한 데이터를 `App.jsx`를 통해 전역적으로 전달하려고 했습니다.  
하지만 여러 팀원의 데이터와 흐름이 한 파일에 모이면서 데이터가 겹치거나 구조가 복잡해지는 문제가 발생했습니다.

이를 해결하기 위해 `1:1 문의` 페이지에서는 단순 라우터 이동 방식 대신,  
`useState`와 `createContext`를 활용해 페이지 전환과 데이터 전달을 처리했습니다.  
그 결과 필요한 데이터만 독립적으로 관리할 수 있었고, 다른 팀원의 작업과 충돌하지 않도록 개선할 수 있었습니다.

## 🔧 개선 예정

<br />

## 🌱 배운 점

- CRUD 기능은 이전까지 가장 어렵게 느꼈던 영역이었는데, 이번 프로젝트를 통해 직접 구현해보며 흐름을 더 구체적으로 이해할 수 있었습니다.
- 검색, 토글, 페이지네이션, 작성/수정/삭제 기능을 연결하면서 게시판 구조를 실제로 다뤄볼 수 있었습니다.
- 팀 프로젝트에서는 단순히 기능을 구현하는 것뿐 아니라, 데이터 전달 방식과 역할 분리 역시 중요하다는 점을 배울 수 있었습니다.
- 공용 컴포넌트와 상태 관리 방식을 고민하면서 재사용성과 구조 설계의 필요성을 느꼈습니다.

<br />

## ⚠️ 아쉬운 점

- 초기 데이터 전달 구조를 조금 더 명확하게 설계했다면 `App.jsx`에 의존하지 않고 더 안정적인 흐름으로 구현할 수 있었을 것 같습니다.
- 게시판 기능을 구현하는 과정에서 상태 관리와 구조를 조금 더 일관되게 정리했다면 유지보수성이 더 높아졌을 것이라고 느꼈습니다.
- 추후에는 1:1 문의 관련 기능을 공통 로직으로 더 정리해 재사용성을 높여보고 싶습니다.

<br />

## 📚 프로젝트 후기

CRUD 기능은 이번 프로젝트에서 제가 가장 어렵게 느꼈던 부분이었기 때문에,  
오히려 직접 맡아서 구현해보고 싶다는 생각으로 게시판 기능을 담당하게 되었습니다.  
쉽지 않은 작업이었지만, 작성 / 수정 / 삭제 흐름을 직접 완성해보며 큰 성취감을 느꼈고,  
이 과정을 통해 이전보다 한층 더 깊이 이해할 수 있었습니다.  
앞으로도 어렵게 느껴지는 기능일수록 직접 부딪혀보며 성장하고 싶다고 느꼈습니다.
